# EasyNovel 功能说明

EasyNovel 是一个本地运行的长篇小说生成工具。它的目标不是让 AI 一次请求写完整本书，而是在用户点击一次生成后，由后台按章节自动循环生成、检查、修订、更新记忆，最后合并为完整 Markdown 小说。

## 1. 启动方式

在项目目录运行：

```powershell
python app.py
```

默认访问地址：

```text
http://127.0.0.1:8000
```

本地模拟模式：

```powershell
$env:NOVEL_MOCK="1"
python app.py
```

模拟模式不会调用真实 AI，适合测试页面和生成流程。

## 2. 主页面功能

主页面地址：

```text
http://127.0.0.1:8000/
```

### 2.1 AI 设定助手

主页面顶部提供 `AI 设定助手`：

- 输入小说想法、题材、主角、世界观、章节数、风格和禁忌。
- 点击 `生成设定` 后，系统会调用当前选择的模型配置。
- AI 会返回可导入的 `故事大纲`、`人物设定`、`章节目录` 和 `写作风格`。
- 点击 `一键导入到设定页` 后，会填入主页面对应输入框并保存。
- 点击 `清空对话` 可以重置设定助手。

### 2.2 小说设定输入

主页面提供四个核心输入区：

- `故事大纲`：填写全书主线、冲突、转折、结局方向、不能提前揭露的秘密。
- `人物设定`：填写人物身份、性格、动机、关系、秘密、禁止改变的设定。
- `章节目录`：填写章节标题、每章必须发生的事件、禁止事项。
- `写作风格`：填写文风、叙事节奏、视角、语言偏好等。

点击 `保存设定` 后，这些内容会写入：

- `input/outline.txt`
- `input/characters.txt`
- `input/chapters.txt`
- `input/style.txt`

### 2.3 模型配置选择

主页面不直接填写 API Key。需要先进入 `配置 API` 页面保存模型配置，然后在主页面的 `模型配置` 下拉框中选择一个已保存配置。

主页面会显示当前配置的：

- 接口风格
- 模型名称
- 推理深度

### 2.4 生成参数

主页面支持三个生成参数：

- `每章目标字数`：传给 AI 的章节目标字数。
- `自动修订次数`：章节检查不通过时自动修订的次数。
- `携带最近章节数`：生成新章节时额外带入最近 N 章正文，默认 3。
- `创作自由度`：对应接口请求里的 `temperature`。

这些生成参数会保存在浏览器本地存储中。`每章目标字数` 上限为 20000，旁边的 `最大` 按钮可以一键填入上限。

### 2.5 开始生成

点击 `开始生成` 后，后台会按章节目录逐章执行：

```text
生成当前章节
检查是否偏离大纲或人设
必要时自动修订
保存单章 Markdown
更新前文记忆
更新人物状态
进入下一章
```

全部章节完成后，会合并成：

```text
output/novel.md
```

每章单独输出到：

```text
output/chapter-001.md
output/chapter-002.md
...
```

### 2.6 暂停

点击 `暂停` 后，任务会在当前章节处理完成后停止。它不是强制中断当前 API 请求，而是请求后台在章节边界停止。

### 2.7 进度显示

进度区域显示：

- 当前生成到第几章
- 总章节数
- 当前任务状态
- 最近生成日志

### 2.8 输出功能

`查看成稿`：

- 读取 `output/novel.md`
- 显示到页面底部的 `成稿预览`
- 如果还没有生成成稿，会提示先开始生成

`下载 Markdown`：

- 下载完整小说 `novel.md`
- 如果还没有生成成稿，会提示先开始生成

## 3. API 配置页面

配置页面地址：

```text
http://127.0.0.1:8000/config.html
```

进入方式：主页面右上角点击 `配置 API`。

### 3.1 配置字段

配置页支持填写：

- `配置名称`：保存后的配置名称，主页面通过它选择模型。
- `接口风格`：选择 API 请求格式。
- `Base URL`：接口根地址。
- `模型`：模型名称。
- `API Key`：接口密钥。
- `全局 Prompt`：当前模型配置的最高优先级提示词，所有 API 调用都会首先遵守它。
- `最大输出 token`：单次请求允许的最大输出。
- `超时秒数`：单次请求超时时间。
- `推理深度`：无、低、中、高。
- `Anthropic Version`：Anthropic/Authpic 风格接口使用。

配置会保存在浏览器本地存储中，方便个人本地使用。

## 4. 支持的 API 风格

### 4.1 Chat Completions

适用于 OpenAI 兼容的：

```text
/chat/completions
```

请求特点：

- 使用 `messages`
- 使用 `Authorization: Bearer <API Key>`
- 推理深度会映射为 `reasoning_effort`

### 4.2 Responses

适用于 OpenAI 兼容的：

```text
/responses
```

请求特点：

- 使用 `input`
- 使用 `Authorization: Bearer <API Key>`
- 推理深度会映射为 `reasoning.effort`

### 4.3 Anthropic / Authpic

适用于 Anthropic Claude 风格的：

```text
/messages
```

请求特点：

- 使用 `x-api-key`
- 使用 `anthropic-version`
- 推理深度会映射为 `thinking.budget_tokens`

## 5. 配置页操作

### 5.1 保存配置

点击 `保存配置` 后，当前配置会保存到浏览器本地存储。

主页面会自动读取这些配置，用于生成时选择。

### 5.2 测试连接

点击 `测试连接` 后，后端会用当前配置发送一个很短的测试请求。

用途：

- 检查 API Key 是否可用
- 检查 Base URL 是否正确
- 检查模型名是否可调用
- 检查接口风格是否匹配

### 5.3 获取模型

点击 `获取模型` 后，后端会请求：

```text
/models
```

如果接口支持模型列表，会把模型显示在状态区域。点击模型名可以填入 `模型` 输入框。

部分第三方接口可能不支持 `/models`，这时需要手动填写模型名。

## 6. 生成流程细节

每一章生成时都会带上：

- 故事大纲
- 人物设定
- 写作风格
- 完整章节目录
- 已发生剧情摘要
- 最近 3 章正文，具体数量可在主页面修改
- 当前人物状态
- 当前章节标题
- 当前章节要求

这样可以尽量减少：

- 跑题
- 人设崩坏
- 提前剧透
- 漏写章节事件
- 前后矛盾

## 7. 自动检查与修订

每章生成完成后，系统会再调用一次 AI 检查：

- 是否偏离故事大纲
- 是否违反人物设定
- 是否提前剧透后续剧情
- 是否漏掉本章必须事件
- 是否和前文摘要或人物状态矛盾

如果检查不通过，会按照 `自动修订次数` 尝试修订。

## 8. 记忆文件

生成过程中会自动维护两个记忆文件：

```text
memory/summary.txt
memory/character-state.txt
```

`summary.txt` 用于保存前文剧情、时间线、线索、伏笔。

`character-state.txt` 用于保存人物当前处境、动机、关系变化、掌握的信息、物品、秘密等。

后续章节会读取这些内容，保证长篇连贯。

## 9. 输出文件

生成后的文件包括：

```text
output/chapter-001.md
output/chapter-002.md
output/chapter-003.md
...
output/novel.md
```

`chapter-xxx.md` 是每章单独成稿。

`novel.md` 是整本小说合并后的 Markdown 文件。

## 10. 后端接口

当前后端提供这些接口：

- `GET /api/project`：读取故事设定。
- `POST /api/project`：保存故事设定。
- `GET /api/memory`：读取剧情记忆和人物状态。
- `POST /api/test-config`：测试当前 API 配置。
- `POST /api/models`：获取模型列表。
- `POST /api/plan`：根据对话生成可导入的小说设定。
- `POST /api/generate`：创建生成任务。
- `GET /api/status/<job_id>`：查询任务进度。
- `POST /api/stop/<job_id>`：请求暂停任务。
- `GET /api/output`：读取完整成稿。

## 11. 安全与注意事项

- API Key 保存在浏览器本地存储中，适合个人本地使用。
- 不要把浏览器本地存储、配置页面截图、包含 API Key 的内容发给别人。
- `input/` 文件不会保存 API Key。
- `output/` 文件只保存生成内容。
- 长篇生成会产生多次 API 调用，章节越多、自动修订次数越多，消耗越高。
- 真实生成前建议先用少量章节测试配置和效果。

## 12. 推荐章节目录格式

推荐每章写清楚必须事件和禁止事项：

```text
第1章：雨夜来信
必须写：林舟收到母亲失踪前留下的信，决定回到旧城。
不能写：不能揭露母亲失踪真相。

第2章：旧城旅馆
必须写：林舟入住旧城旅馆，遇到沈青璃。
不能写：沈青璃不能直接表明身份。
```

越明确的章节要求，越容易控制 AI 不跑题。
